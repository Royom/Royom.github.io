1.模仿、探索与自我提升：慢思考推理系统的复现之路

Why: 工业界关于慢思考推理系统技术保密，作者在尝试完成一个推理系统

What: 提出了一个“模仿、探索与自我提升”的三阶段训练框架，旨在模仿思维链中的思考模式。

模仿：用长思维链数据对模型进行了微调，目的在于让大模型在回答问题之前内部生成详细的推理步骤#真的能吗？内部真的是在做推理？#。看着有Prompt模板

探索：采用简单的搜索策略，对问题生成多个解答轨迹，直到找到包含正确答案的解答（说的比较宽泛）

自我提升：采用了监督微调（自监督？）；采用直接偏好优化（DPO）#第一次听说#

Direct Preference Optimization: Your Language Model is Secretly a Reward Model简单了解了一下，这是23年提出的一个方法，相比RLHF，计算量更轻，无需拟合奖励模型，也无需在微调期间从LM采样或执行显著的超参数调整。值得注意的是，DPO 在情绪控制的能力上超越了 RLHF，提高了总结和单轮对话的响应质量，同时大大简化了实现和训练。

7.A Comprehensive Survey of Retrieval-Augmented Generation (RAG): Evolution, Current Landscape and Future Directions

RAG 2024.10的综述

1.预备知识：RAG两个关键组件：检索器和生成器

2.当前的技术：

  1. 检索机制：BM25（完善的老方法，对关键词匹配很有效，在理解语义方面有局限性）、DPR（密集通道检索，擅长捕获查询和文档之间的语义相似性）、REALM（检索增强语言模型，即利用LLMS来帮助判断什么更好）
  2. 生成器机制：T5（Text-to-Text Transfer Transformer），BART（idirectional and Auto-Regressive Transformer) 两种模型，暂时不细究
  3. 多模态的一些相关新研究，如果确定做RAG再回头看

3.当前的挑战：

  1. 检索机制虽然强大，但仍然难以检索最相关的文档，特别是在处理不明确的查询或专业知识领域时。
  2. 检索和生成之间的集成虽然在理论上是无缝的，但在实践中有时会失败。例如，生成模块可能并不总是有效地将检索到的信息合并到其响应中，导致检索到的事实和生成的文本之间不一致或不连贯。
  3. RAG 模型的计算开销也是一个问题，因为它们需要为每个查询执行检索和生成步骤。这种双重过程可能会占用大量资源，特别是对于大规模应用而言
  4. 道德风险，可能放大检索来源中存在的偏见

4.当前研究的方向：

  1. 需要努力改进检索技术，包括结合更复杂的查询扩展和上下文消歧，以提高这些领域的性能。
  2. 研究更好的对齐机制，例如改进的注意力模型或分层融合技术

四.强化性学习

1.KV Cache

最新进展

  1. 缓存压缩与量化：研究者提出了多种方法来压缩 KV 缓存的大小，以减少内存占用。例如，CacheGen 使用自定义张量编码器对 KV 缓存进行压缩和流式处理，从而加快 LLM 系统的上下文加载速度。 ACM Digital Library
  2. 自适应缓存压缩：一些方法根据模型的内部结构，自适应地压缩 KV 缓存。例如，Adaptive KV Cache Compression 方法通过分析注意力模块的结构，选择性地丢弃或压缩特定层的缓存，以减少内存占用。arXiv
  3. 低秩空间注意力：Eigen Attention 方法在低秩空间中执行注意力操作，从而减少 KV 缓存的内存需求。ACL Anthology
  4. 关键令牌选择：Keyformer 方法在推理时选择关键令牌，以减少 KV 缓存的大小，从而提高推理效率。MLSys Proceedings
  5. 量化技术：KIVI 方法提出了一种无须调优的非对称 2 位量化技术，用于减少 KV 缓存的内存占用，同时保持模型性能。arXiv Atom: Low-bit Quantization for Efficient and Accurate LLM Serving 低位量化方法 未看
  6. KV Cache Compression, But What Must We Give in Return? A Comprehensive Benchmark of Long Context Capable Approaches 这是一篇有关评估长上下文能力的基准的论文，这里面指出了当前的一些KV优化方法：只关注了它提到的KV Cache最新优化方法（可以再看） token dropping, prompt compression, linear-time sequence models, and hybrid architectures

面临的挑战

  • 长上下文处理：随着上下文长度的增加，KV 缓存的大小线性增长，导致内存瓶颈，影响推理效率。
  • 批处理大小限制：在大批处理情况下，KV 缓存的内存需求显著增加，限制了模型的并行处理能力。
  • 缓存管理策略：如何有效地管理和压缩 KV 缓存，以在减少内存占用的同时，保持模型性能，是一个关键问题。

代表性的优化方式/架构

  1. MiniCache：通过在深度维度上压缩 KV 缓存，减少内存占用，同时保持模型性能。 arXiv
  2. 混合精度量化：通过对重要的 KV 对采用高精度表示，次要的采用低精度表示，实现缓存压缩与性能的平衡。 arXiv
  3. 自适应缓存合并：根据模型的需求，自适应地合并或丢弃部分 KV 缓存，以减少内存占用。 arXiv](https://arxiv.org/abs/2407.08454?utm_source=chatgpt.com)
  4. CXL 内存扩展：利用 Compute Express Link（CXL）技术，将 KV 缓存存储在扩展内存中，以缓解内存压力。 MLforSystems

正在研究的方向

  • 缓存压缩算法：开发更高效的压缩算法，以在最小化内存占用的同时，保持或提升模型性能。
  • 硬件加速：探索专用硬件或硬件加速技术，以提高 KV 缓存管理和访问的效率。
  • 动态缓存管理：研究动态调整 KV 缓存大小和策略的方法，以适应不同的输入和任务需求。
  • 跨层信息共享：探索在不同层之间共享或重用 KV 缓存的可能性，以减少冗余存储。

我现在的个人理解：KV Cache是在注意力模块中，进行QKV矩阵计算时为了加快，将KV缓存到显存里面的一种方法。

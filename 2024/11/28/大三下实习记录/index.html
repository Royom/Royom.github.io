1.2 学习路线

  1. 部署和运行项目：
    • 克隆项目代码，安装依赖（如 Docker、Python 环境）。
    • 阅读项目的 README.md 文件，理解架构图和主要组件。
    • 使用 docker-compose 启动服务，观察服务间的交互日志。
  2. 学习技术基础：
    • Kafka: 学习如何创建 Topic，生产者/消费者模式，查看消息流（例如使用 Kafka Console）。
    • Spark: 熟悉 Spark 的核心概念（RDD、DataFrame、Streaming）以及如何使用 Python 编写 Spark 应用。
    • Cassandra/PostgreSQL/S3: 学习数据存储与查询的基本操作。
    • Docker: 掌握 Docker 容器化技术和 docker-compose 管理多服务的能力。
  3. 改进与扩展：
    • 修改数据生成器以生成更贴合现实场景的数据。
    • 在 Spark 数据处理脚本中添加自定义逻辑（如实时数据聚合或清洗规则）。
    • 部署到本地 Kubernetes 集群（如使用 Minikube），体验容器编排和资源调度。
  4. 深度优化与总结：
    • 使用监控工具（如 Prometheus、Grafana）分析 Kafka 和 Spark 的性能瓶颈。
    • 编写文档和博客，总结项目的技术实现和个人学习收获。

1.3 重点关注：

深入阅读和分析项目代码的实现。重点关注：

  • 数据是如何从Kafka流入Spark，以及在Spark中如何进行处理的。
  • 数据处理后的结果是如何存储到Cassandra中的。
  • 项目是否涉及到错误处理、数据清洗、实时处理优化等内容。

1.2.1组件之间的关系：

  1. Kafka 系统
    • Zookeeper 为 Kafka Broker 提供元数据存储。
    • Broker 与 Schema Registry 通信，管理消息模式。
    • Control Center 与 Broker 和 Schema Registry 通信，用于监控。
  2. 支持工具
    • Airflow 依赖 PostgreSQL 提供数据库支持。
    • Spark Worker 依赖 Spark Master 进行任务协调。
    • Cassandra 独立存在，可供其他服务使用。

适合你的理由: 该项目基于 Python 和分布式系统技术，能让你接触到 Kafka 和 Spark 的实际应用，同时熟悉如何使用 Docker 管理分布式服务。

选项解释：

  • -p 8080:8080：将容器内部的 8080 端口映射到主机的 8080 端口，以便通过 http://localhost:8080 访问 Web 项目。
  • --name test：为容器命名为 test，便于管理。
  • -v D:/test:/app：将本地目录 D:/test 挂载到容器内的 /app 目录，用于共享代码或配置文件。
  • --network test-net：将容器加入 test-net 网络，与 Redis 容器通信。
  • -d：后台运行容器。
  • test:v1：指定使用的镜像版本为 v1。
